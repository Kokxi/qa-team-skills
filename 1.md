## 二、提交前准备

### 2.1 项目结构检查清单

提交前确保技能项目满足以下条件：

```
qa-team-skills/
├── SKILL.md              ← 必须：技能入口文件，含 YAML frontmatter
├── README.md             ← 强烈推荐：项目说明文档
├── LICENSE               ← 强烈推荐：开源许可证（建议 MIT）
├── VERSION               ← 推荐：版本号文件
├── prompts/              ← 推荐：指令 Prompt 文件
├── examples/             ← 加分项：使用示例
├── docs/                 ← 加分项：详细文档
└── team/                 ← 加分项：配置文件
```

### 2.2 SKILL.md 规范要求

所有平台都要求 SKILL.md 包含合法的 YAML frontmatter：

```yaml
---
name: qa-team-skills               # 必填，小写字母数字+连字符，1-64字符
description: >                      # 必填，1-1024字符，足够具体以便 AI 匹配
  团队统一技能——将 AI Agent 嵌入软件测试核心环节。
  6 大指令覆盖：需求评审、用例设计、AI Agent 专项测试、缺陷分析、报告生成、团队管理。
metadata:                          # 可选
  type: user
  version: v1.2.0
  author: QA Team
compatibility:                     # 可选，声明兼容性
  - claude-code
  - cursor
  - opencode
license: MIT                       # 可选，开源许可证
---
```

### 2.3 FST 规范（部分平台需要）

部分平台（如 BringSkills）要求遵循 Agent Skills 规范（[agentskills.io](https://agentskills.io)），核心要求：

- 每个技能一个独立文件夹
- 文件夹名 = 技能名（kebab-case）
- 内含 `SKILL.md` 作为入口
- 遵循标准 frontmatter 字段

## 三、ClawHub（OpenClaw 官方）

**地址**：https://clawhub.ai  
**定位**：OpenClaw 官方 Skill 注册中心，生态最大、用户最集中，**首选提交平台**。  
**收录规模**：5 万+ 技能，日均调用千万级。

### 3.1 提交方式一：CLI 提交（推荐）

```bash
# 1. 安装 ClawHub CLI
npm install -g @openclaw/cli

# 2. 登录（需要注册 ClawHub 账号）
claw login

# 3. 提交技能（在项目根目录执行）
claw skill publish \
  --name qa-team-skills \
  --path ./ \
  --description "为测试团队设计的统一 AI 辅助能力——6 个标准化指令覆盖需求评审到团队管理" \
  --tags "testing,qa,test-case,bug-analysis,report" \
  --categories "development,testing"

# 4. 查看提交状态
claw skill status qa-team-skills
```

### 3.2 提交方式二：Web 提交

1. 打开 https://clawhub.ai
2. 注册/登录账号
3. 点击右上角 **Publish Skill** 按钮
4. 填写表单：
   - **Skill Name**：`qa-team-skills`
   - **Category**：选择 `Development` → `Testing`
   - **Description**：简要描述技能功能
   - **GitHub URL**：填写本项目的 GitHub 仓库地址
   - **Tags**：`testing`, `qa`, `test-case`, `bug-analysis`, `test-report`, `ai-testing`
5. 提交审核

### 3.3 审核周期

- AI 自动审核：即时（检查格式安全）
- 人工审核：1-3 个工作日
- 通过后自动上线，出现在 ClawHub 搜索结果中

## 四、腾讯云 SkillHub

**地址**：https://skillhub.tencent.com  
**定位**：腾讯云专为中国用户优化的 AI Skills 社区，国内访问速度快，收录 3.3 万+ Skills。  
**适用场景**：面向国内测试团队推广的首选平台。

### 4.1 提交方式

```bash
# 方式一：GitHub PR 提交
# 1. Fork 官方收录仓库
# 2. 在仓库的 skills/ 目录下添加你的技能目录
# 3. 提交 PR，等待审核

# 方式二：后台直接提交（需注册腾讯云账号）
```

**Web 提交流程**：

1. 打开 https://skillhub.tencent.com
2. 注册腾讯云账号并登录
3. 点击 **提交技能**
4. 填写：

## 五、SkillsMP

**地址**：https://skillsmp.com/zh（中文版）  
**定位**：全球最大的社区驱动技能市场，收录 170 万+ 开源技能，自动从 GitHub 同步。  
**优势**：无需手动提交，GitHub 公开仓库自动收录。

### 5.1 自动收录条件

SkillsMP 通过爬虫自动索引 GitHub 上的技能仓库。要确保被收录：

1. **仓库公开**：确保 GitHub 仓库为 public
2. **SKILL.md 有效**：根目录或 `.claude/skills/*/SKILL.md` 格式正确
3. **描述准确**：`description` frontmatter 字段要足够具体
4. **等待爬取**：SkillsMP 定期扫描 GitHub，通常 1-7 天内自动收录

### 5.2 手动提交

```bash
# 通过 SkillsMP API 提交
curl -X POST https://skillsmp.com/api/v1/skills \
  -H "Content-Type: application/json" \
  -d '{
    "name": "qa-team-skills",
    "repository": "https://github.com/Kokxi/qa-team-skills",
    "description": "为测试团队设计的统一 AI 辅助能力——6 个标准化指令覆盖需求评审到团队管理",
    "tags": ["testing", "qa", "test-automation"]
  }'
```

### 5.3 提交后管理

- 在 SkillsMP 注册账号后，可认领自己的技能
- 认领后可更新描述、分类、标签
- GitHub 仓库更新后，SkillsMP 会自动同步（延迟 24-48 小时）

## 十五、通用 Tips

### 15.1 SKILL.md 优化建议

- **名称唯一性**：在所有平台中保持 `qa-team-skills` 一致，建立品牌认知
- **描述精准**：前 100 个字符最关键（搜索结果摘要），使用高价值关键词
- **Tag 策略**：覆盖中英文关键词——`testing`, `qa`, `test-case`, `bug-analysis`, `test-report`, `ai-testing`, `测试`, `测试用例`, `缺陷分析`, `测试报告`

### 15.2 提交优先级推荐

```
第一梯队（必须提交）：
  1. ClawHub          → 最大生态，核心用户来源
  2. 腾讯云 SkillHub  → 国内用户访问最快
  3. SkillsMP         → 自动收录，零成本
  4. LobeHub          → OpenCode 集成

第二梯队（推荐提交）：
  5. Skills.sh        → 排名曝光和数据分析
  6. Open Agent Skills → 开源社区覆盖
  7. SkillHub 中国    → 国内镜像加速

第三梯队（可选提交）：
  8. Killer-Skills    → 安全认证背书
  9. ZeroOne 市场     → 质量认证（审核周期长）
  10. BringSkills     → 商业化（如无收费计划可跳过）

被动收录（无需操作）：
  - Agent Skills Hub  → 自动爬取
  - SkillsMP          → 自动同步 GitHub
```

### 15.3 提交后维护

- **版本同步**：技能更新后，在各平台同步更新描述和版本号
- **用户反馈**：监控各平台的评论/评分，及时回复和修复
- **数据跟踪**：定期检查各平台的下载量数据，优化投放策略
- **GitHub 保持活跃**：大多数平台以 GitHub 仓库为数据源，保持仓库更新

### 15.4 网络问题处理

国内访问部分海外平台（ClawHub / SkillsMP / Skills.sh 等）可能需要代理：

```bash
# Git 代理配置（如已有代理）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 仅对 GitHub 使用代理
git config --global http.https://github.com.proxy http://127.0.0.1:7890
```
